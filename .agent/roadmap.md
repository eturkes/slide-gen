# slide-gen — roadmap

Goal: a CLI + home for slides across `~/Projects/*`. Discover sibling projects → toggle on/off → `codex exec` generates a `deck.html`-style deck per enabled project → render to PDF. The repo is the deck home: `templates/deck.html` = style contract; `decks/<project>/` = generated output.

Stack: **MoonBit (native backend)** — the agent-oriented bet (AGENTS.md ethos). UX = flag subcommands (no TUI). State = gitignored local. Discovery = all sibling dirs, default-off. (Owner-confirmed at M2 plan.)

Seed: commit `0526d53` (repo init: deck.html, agent guidance, `.agent` boilerplate) + the `templates/deck.html` move. `deck.html` = the content-stripped generalization of the proven prototype `~/Projects/spine-segmentation/docs/slides/` (filled deck.html + figures.py + render.py → PDF) — M2/M3 mirror it (mechanics in memory.md).

## M2 — Generation (REVIEWED)

As-designed: `slide-gen generate [<project>]` drives `codex exec` (one shot per project) to author a deck BUNDLE at `decks/<project>/` — `deck.html` (template-styled) + `provenance.json` (per-claim source citations) + `gaps.md` (unverifiable / human-needed) + `figures.py` + `assets/*.png`. A deterministic MoonBit VALIDATOR gates every bundle (template conformance + provenance quote-grep + furigana structural guards); a project-local Python analyzer cross-checks furigana readings (hybrid). Deck CONTENT = codex's job (nondeterministic, env-gated + external `timeout`); the harness + validator are the testable MoonBit surface.

### Decisions (owner + research, at plan; env/tool mechanics → memory.md)
- **Gate PASSED** — `codex exec` with an explicit `--sandbox` IGNORES the interactive trust gate (probe-verified: read-only exec on a non-trusted sibling → exit 0, replied); `trust_level` only gates project-local `.codex/` loading, not runtime.
- **Sandbox** = `--dangerously-bypass-approvals-and-sandbox` (owner; the container is externally sandboxed). Harness must still direct writes to the bundle dir (bypass = no write confinement).
- **Input scope** = codex may read ANYTHING (owner; no deny-list / no input curation). The provenance **quote-grep** anti-hallucination check stays independent of that.
- **Furigana** = HYBRID — codex writes the three tiers incl. furigana → MoonBit structural guards (`.jp`==rt-stripped-`.jpf` · `<rt>` all-kana · ruby covers kanji runs) → Python analyzer (fugashi+UniDic / SudachiPy) independently re-reads the JA; disagreements / analyzer-unknowns → `gaps.md`.
- **Output** = COMMITTED deck home — track `decks/<project>/{deck.html,provenance.json,gaps.md,figures.py,assets/*.png}`; gitignore only render intermediates (`_deck_full.png`, `page_*.png`). **Privacy**: the no-secrets/PHI rule covers the WHOLE committed bundle (quote-grep COPIES verbatim source text into `provenance.json`, so a cited secret would leak). No deterministic secret gate (owner chose read-anything + committed) → prompt guideline + human review before the owner pushes (owner handles remote).
- **Provenance contract**: `provenance.json` rows `{id,slide,claim,status:verified|inferred|gap,src:"path:Lstart-Lend" (parent-relative or absolute),quote:"<verbatim>"}`; deck claims carry `data-claim="id"`. The quote must LITERALLY occur WITHIN the cited `src` line-slice (a cheap EXISTENCE check — kills invented quotes; NOT a truth check — misattribution / cherry-pick still pass → analyzer + human). A row must NOT cite a generated bundle file (no self-citation). Prompt drives a codex DRAFT→re-verify-quotes+line-numbers→FINAL self-critic.
- **Section-count gotcha**: count `<section class="slide">` after COMMENT-STRIP — the template's line-8 header comment inflates a raw `grep -c` by 1.
- **Figures**: codex authors `figures.py` from COMMITTED, CITED numbers and runs it (bypass permits exec) → `assets/*.png`; raw-data or failed figures → a `.fig-ph` placeholder + a `gaps.md` entry (human runs it, à la the prototype's `figure_example.py`). Each committed `assets/*.png` earns a `provenance.json` row citing the real number-SOURCES it plots (NEVER `figures.py` — a bundle file is self-citation). figures.py + assets are CONDITIONAL: a text-only deck has neither → `validate_bundle_dir` requires only deck.html + provenance.json + gaps.md, checking figures.py/assets solely when the deck computes figures.

### Review closure
Exhaustive commit/current-tree review accepted and fixed: unified context-aware HTML cleaning + stray-`<`-safe/full-name tag walks (quoted comment/raw-text/doctype/close decoys no longer hide claims; entity-encoded kanji cannot bypass ruby coverage); stricter furigana close-state; provenance positive-slide/nonempty schema + unique IDs + exact bidirectional deck parity + slide ownership; `gap`/`no gaps` consistency; canonical-source confinement with a direct-symlinked-project allowed-root model; closed no-follow bundle/figure layout; tracked-write tripwire on every producer/postprocessor exit and every tracked path, with a HEAD-diff fingerprint for already-dirty files; per-project batch baselines; real reserved stage/backup parents; rollback-safe/restart-recoverable publication via `decks/.previous/`; both live figures reproducible on the installed Chromium surface without undeclared Matplotlib. Gates: 114 MoonBit + 7 Python tests, 0-warn; both committed bundles structurally/cross-reference revalidated, with rehab quotes rechecked against its live sibling (the spine source sibling is absent, so only its source-dependent quote pass was unavailable); figures byte-reproduced; privacy/identifier scan + full-deck visual QA clean.

Accepted limits (owner decisions / platform boundary): bypassed Codex can still write to sibling repos, untracked/ignored paths, non-git paths, outside the repo, or while the Git snapshot is unavailable — the in-repo tracked tripwire is defense-in-depth, not a sandbox. Provenance `realpath` confinement is snapshot-based (no `openat` handle chain), so a concurrent hostile filesystem actor could race it. Directory publication uses portable renames: ordinary failures roll back and the next run restores an interrupted backup, but a crash between old→backup and stage→public temporarily removes the public path until recovery. Quote-grep proves literal existence in the cited slice, never truth/support.
