## MoonBit reference (native; pre-1.0 — verified at M1.1)
- Install: `curl -fsSL https://cli.moonbitlang.com/install/unix.sh | bash` → `~/.moon/bin` (moon, moonc, moon-lsp, moonrun…); NOT on non-interactive PATH → `export PATH="$HOME/.moon/bin:$PATH"` per Bash call, or drive via `just` (justfile prepends it). Pin via `MOONBIT_INSTALL_VERSION=<v>` or installer arg 1 (default `latest`); record exact via `moon version --all`. Native needs cc (build-essential ✓, x86_64).
- Config = DSL by default (flags rr_moon_mod/rr_moon_pkg on): `moon.mod` TOML-ish (`name="user/proj"` needs the slash; `preferred_target="native"`; module deps `import { "pkg@ver", }`); per-pkg `moon.pkg` (`import { "mod/pkg" @alias, }`; `supported_targets="+native"`; `options("is-main": true)`). Unused module dep = no warning. `moon fmt` blank-lines moon.mod fields + reflows calls. Core pkgs (argparse…) imported by path, bundled (no `moon add`).
- Layout: `moon new` → root lib pkg + `cmd/main` exe (NOT `src/main`); keep logic in lib, main thin. Output `_build/`; native ELF = `_build/native/{debug,release}/build/cmd/main/main.exe` (`.exe` on Linux). `.mooncakes/` = fetched deps (gitignored, readable for API ref via `examples/` + `*.mbti`). `moon install` deprecated → `moon build`/`check` auto-fetch.
- async (`moonbitlang/async@0.20.0`; native/LLVM Linux epoll): `async fn main` needs BOTH a `moonbitlang/async` package import (in the exe's `moon.pkg`, NOT just a source `@async`) AND a real suspension, else `unused_async` warns; `@fs` adds the `moonbitlang/async/fs` import → M1.1 sync; M1.2 flipped to `async fn main` (`moonbitlang/async` in cmd/main, `moonbitlang/async/fs` in root lib) at the first `@fs` await. async main may call raising fns directly (error boundary); sync `fn main` cannot → `try EXPR catch { e => … } noraise { v => … }` (`try?` deprecated). Concurrency = `with_task_group`. `@process.collect_output_merged → (Int, &@io.Data)`; `@fs.{read_file, write_file(p, content, create_mode=CreateOrTruncate), readdir(p, include_hidden?, include_special?, sort?), kind(p, follow_symlink?)→FileKind, tmpdir(prefix~), mkdir, symlink(linkpath, target~), rmdir(p, recursive=true)}`. `kind` on FULL `parent/name` (bare resolves vs cwd), FOLLOWS symlinks by DEFAULT (`follow_symlink=false`=lstat) + raises on unresolvable target → ONE follow-stat classifies dir / symlink→dir (candidate) vs broken / non-dir, so the planned lstat+stat 2-call split was redundant. write content: String IS `@io.Data` (`""`=empty file); `&@io.Data`≠String on READ → `.text()`/`.binary()`. `rmdir(recursive=true)` unlinks symlinks (no traverse). `defer` REJECTS async/raising calls → cleanup via `@async.with_task_group() <| group => { group.add_defer(() => <async cleanup>) }`. `@env.current_dir()→String?` (core/env) for cwd. `x/fs.is_dir` follows symlinks + no lstat → use `@fs.kind`.
- `@argparse` (clap-like, subcommands CONFIRMED): `@argparse.Command(name, about?, flags?, options?, positionals?, subcommands?, version?, arg_required_else_help?, subcommand_required?, …)` (`::new` deprecated → call `Command(...)`). `cmd.parse(argv? : ArrayView[String], env? : Map[String, String]) -> Matches raise` (test a dispatch key via `cli().parse(argv=["list"]).subcommand`; an `Array` literal coerces to the view) auto-prints+exits on `-h/--help` (and `--version` only when `version=` is set on the `Command`), raises on bad args (msg is `Show` → `println(err)`, currently stdout — M1.3 routes to stderr + nonzero exit); no-arg = real process args. `Matches.subcommand : (String, Matches)?`. `cmd.render_help() -> String` for goldens. `runtime_exit_native` not publicly callable → need another exit for nonzero codes.
- Tests: `inspect(v, content=…)` snapshot, fill/update via `moon test --update`; `json_inspect(arr, content=[...])` for arrays. Multiline `#|` only in `let`/parens, NOT as a bare `content=` arg (warns) → single-line goldens use plain quoted strings. `inspect` on a tuple warns (Show-for-debug deprecated) → inspect interpolated Strings (`"\{a}|\{b}"`) instead. `_wbtest.mbt` whitebox (sees private), `_test.mbt` blackbox. `async test` requires `moonbitlang/async` in the TEST's scope; `for "test"` import blocks feed BLACKBOX only, NOT whitebox → an async test in a lib whose non-test code doesn't use `@async` must be blackbox (`_test.mbt`) with `moonbitlang/async` in a `for "test"` block. Blackbox refs the ROOT pkg's pub items UNQUALIFIED (the hyphen in `eturkes/slide-gen` yields no valid auto-alias; `@lib` is cmd/main-private, invisible to blackbox). `@quickcheck` = Arbitrary + `samples` gen (no shrinker/fuzzer). `@json` derive ToJson/FromJson; no TOML lib → newline state file.
- `String::compare` / `<` / `>` = SHORTLEX (length first, THEN content), NOT lexicographic → `Array::sort` orders by length. Dictionary order needs `Array::sort_by(cmp)` with an explicit codepoint comparator (`s.to_array()` → compare `Char` by `<`/`>`, shared prefix → shorter first). VERIFIED `"link_to_dir" < "realdir"`=false but `"link" < "real"`=true. `list`/state-file sorting (M1.3) must use this comparator, not the default. Deprecations (0.10.x): `not(x)`→`!x`; `StringView::to_string`→`.to_owned()`; `String::substring(start?,end?)`→slice `s[a:b]` (returns StringView).
- Serena: no MoonBit LSP (official `moon-lsp` ships in `~/.moon/bin` but unwired) → nav via grep + `moon check`/`mooninfo`; core APIs from `~/.moon/lib/core/<pkg>/pkg.generated.mbti` + READMEs.
