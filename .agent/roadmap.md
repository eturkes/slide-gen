# slide-gen ‚Äî roadmap

Goal: a CLI + home for slides across `~/Projects/*`. Discover sibling projects ‚Üí toggle on/off ‚Üí `codex exec` generates a `deck.html`-style deck per enabled project ‚Üí render to PDF. The repo is the deck home: `templates/deck.html` = style contract; `decks/<project>/` = generated output (later milestones).

Stack: **MoonBit (native backend)** ‚Äî the agent-oriented bet (AGENTS.md ethos). UX = flag subcommands (no TUI). State = gitignored local. Discovery = all sibling dirs, default-off. (Decisions confirmed by owner this planning session.)

Seed: commit `0526d53` (repo init: deck.html, AGENTS/CLAUDE, `.agent` boilerplate) + this session's `templates/deck.html` move. `deck.html` is the content-stripped generalization of the **proven prototype** `~/Projects/spine-segmentation/docs/slides/` (filled deck.html + figures.py + render.py ‚Üí PDF) ‚Äî M2/M3 mirror it (mechanics in `memory.md`).

## M2 ‚Äî Generation (planned)

As-designed: `slide-gen generate [<project>]` drives `codex exec` (one shot per project) to author a deck BUNDLE at `decks/<project>/` ‚Äî `deck.html` (template-styled) + `provenance.json` (per-claim source citations) + `gaps.md` (unverifiable / human-needed) + `figures.py` + `assets/*.png`. A deterministic MoonBit VALIDATOR gates every bundle (template conformance + provenance quote-grep + furigana structural guards); a project-local Python analyzer cross-checks furigana readings (hybrid). Deck CONTENT = codex's job (nondeterministic, gated behind an env flag + external `timeout`); the harness + validator are the testable MoonBit surface.

Resolved this plan (probe + research workflow + owner). Env/tool mechanics (codex exec flags, furigana tooling) ‚Üí memory.md; here = decisions:
- **Gate PASSED** ‚Äî `codex exec` with an explicit `--sandbox` IGNORES the interactive trust gate (probe: read-only exec on non-trusted `uas` ‚Üí exit 0, replied). The "20 non-trusted projects" worry is moot; `trust_level` only gates project-local `.codex/` loading, not runtime.
- **Sandbox** = bypass (`--dangerously-bypass-approvals-and-sandbox`) ‚Äî owner; the container is externally sandboxed. Harness must still direct writes to the bundle dir (bypass = no write confinement).
- **Input scope** = codex may read ANYTHING ‚Äî owner (overrode both curated options); no deny-list / no input curation. Independent of that, the provenance **quote-grep** anti-hallucination check stays.
- **Furigana** = HYBRID ‚Äî owner: codex writes the three tiers incl. furigana ‚Üí MoonBit structural guards (`.jp`==rt-stripped-`.jpf` ¬∑ `<rt>` all-kana ¬∑ ruby covers kanji runs) ‚Üí Python analyzer (fugashi+UniDic / SudachiPy) independently re-reads the JA; disagreements / analyzer-unknowns ‚Üí `gaps.md` human-review.
- **Output** = COMMITTED deck home ‚Äî owner: track `decks/<project>/{deck.html,provenance.json,gaps.md,figures.py,assets/*.png}`; gitignore only render intermediates (`_deck_full.png`, `page_*.png`). **Privacy**: the no-secrets/PHI rule covers the WHOLE committed bundle (deck + provenance quotes + gaps + figures + assets), not just the deck ‚Äî quote-grep COPIES verbatim source text into `provenance.json`, so a cited secret would leak. No deterministic secret gate (owner chose read-anything + committed) ‚Üí prompt guideline + human review before the owner pushes (owner handles remote).
- **Provenance contract** (research): `provenance.json` rows `{id,slide,claim,status:verified|inferred|gap,src:"path:Lstart-Lend" (path = parent-dir-relative or absolute),quote:"<verbatim>"}`; deck claims carry `data-claim="id"`. The quote must LITERALLY occur WITHIN the cited `src` line-slice ‚Äî BOTH MUST (quote-in-slice, not merely quote-in-file). This is a cheap EXISTENCE check (kills invented quotes), NOT a truth check ‚Äî misattribution / cherry-pick / wrong-inference still pass (‚Üí analyzer + human). A row must NOT cite a generated bundle file (no self-citation). Prompt drives a codex self-critic DRAFT‚Üíre-verify-quotes+line-numbers‚ÜíFINAL.
- **Section-count gotcha** (memory, re-confirmed): count `<section class="slide">` after COMMENT-STRIP ‚Äî the template's line-8 header comment inflates a raw `grep -c` by 1.
- **Figures**: codex authors `figures.py` from COMMITTED, CITED numbers and runs it (bypass permits exec) ‚Üí `assets/*.png`; raw-data or failed figures ‚Üí a `.fig-ph` placeholder + a `gaps.md` entry (human runs it, √Ý la the prototype's `figure_example.py`). Each committed `assets/*.png` earns a `provenance.json` row citing its `figures.py` + the cited numbers (assets are provenance-covered too).
