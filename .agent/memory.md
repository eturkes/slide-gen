## MoonBit reference (native; pre-1.0 — verified at M1.1)
- Install: `curl -fsSL https://cli.moonbitlang.com/install/unix.sh | bash` → `~/.moon/bin` (moon, moonc, moon-lsp, moonrun…); NOT on non-interactive PATH → `export PATH="$HOME/.moon/bin:$PATH"` per Bash call, or drive via `just` (justfile prepends it). Pin via `MOONBIT_INSTALL_VERSION=<v>` or installer arg 1 (default `latest`); record exact via `moon version --all`. Native needs cc (build-essential ✓, x86_64).
- Config = DSL by default (flags rr_moon_mod/rr_moon_pkg on): `moon.mod` TOML-ish (`name="user/proj"` needs the slash; `preferred_target="native"`; module deps `import { "pkg@ver", }`); per-pkg `moon.pkg` (`import { "mod/pkg" @alias, }`; `supported_targets="+native"`; `options("is-main": true)`). Unused module dep = no warning. `moon fmt` blank-lines moon.mod fields + reflows calls. Core pkgs (argparse…) imported by path, bundled (no `moon add`).
- Layout: `moon new` → root lib pkg + `cmd/main` exe (NOT `src/main`); keep logic in lib, main thin. Output `_build/`; native ELF = `_build/native/{debug,release}/build/cmd/main/main.exe` (`.exe` on Linux). `.mooncakes/` = fetched deps (gitignored, readable for API ref via `examples/` + `*.mbti`). `moon install` deprecated → `moon build`/`check` auto-fetch.
- async (`moonbitlang/async@0.20.0`; native/LLVM Linux epoll): `async fn main` needs BOTH a `moonbitlang/async` package import (in the exe's `moon.pkg`, NOT just a source `@async`) AND a real suspension, else `unused_async` warns; `@fs` adds the `moonbitlang/async/fs` import → M1.1 stayed sync, async + both imports land M1.2 at first `@fs.kind`. async main may call raising fns directly (error boundary); sync `fn main` cannot → `try EXPR catch { e => … } noraise { v => … }` (`try?` deprecated). Concurrency = `with_task_group`. `@process.collect_output_merged → (Int, &@io.Data)`; `@fs.{read_file, write_file(CreateOrTruncate), readdir, kind(path, follow_symlink=false)→FileKind}` — kind on FULL `parent/name` (bare name resolves vs cwd); `&@io.Data` ≠ String → `.text()`/`.binary()`. `x/fs.is_dir` follows symlinks + no lstat → use `@fs.kind`.
- `@argparse` (clap-like, subcommands CONFIRMED): `@argparse.Command(name, about?, flags?, options?, positionals?, subcommands?, version?, arg_required_else_help?, subcommand_required?, …)` (`::new` deprecated → call `Command(...)`). `cmd.parse(argv?, env?) -> Matches raise` auto-prints+exits on `-h/--help` (and `--version` only when `version=` is set on the `Command`), raises on bad args (msg is `Show` → `println(err)`, currently stdout — M1.3 routes to stderr + nonzero exit); no-arg = real process args. `Matches.subcommand : (String, Matches)?`. `cmd.render_help() -> String` for goldens. `runtime_exit_native` not publicly callable → need another exit for nonzero codes.
- Tests: `inspect(v, content=…)` snapshot, fill/update via `moon test --update` (handles `#|` multiline strings). `_wbtest.mbt` whitebox (sees private), `_test.mbt` blackbox. `@quickcheck` = Arbitrary + `samples` gen (no shrinker/fuzzer). `@json` derive ToJson/FromJson; no TOML lib → newline state file.
- Serena: no MoonBit LSP (official `moon-lsp` ships in `~/.moon/bin` but unwired) → nav via grep + `moon check`/`mooninfo`; core APIs from `~/.moon/lib/core/<pkg>/pkg.generated.mbti` + READMEs.
