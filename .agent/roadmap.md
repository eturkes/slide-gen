# slide-gen — roadmap

Goal: a CLI + home for slides across `~/Projects/*`. Discover sibling projects → toggle on/off → `codex exec` generates a `deck.html`-style deck per enabled project → render to PDF. The repo is the deck home: `templates/deck.html` = style contract; `decks/<project>/` = generated output (later milestones).

Stack: **MoonBit (native backend)** — the agent-oriented bet (AGENTS.md ethos). UX = flag subcommands (no TUI). State = gitignored local. Discovery = all sibling dirs, default-off. (Decisions confirmed by owner this planning session.)

Seed: commit `0526d53` (repo init: deck.html, AGENTS/CLAUDE, `.agent` boilerplate) + this session's `templates/deck.html` move. `deck.html` is the content-stripped generalization of the **proven prototype** `~/Projects/spine-segmentation/docs/slides/` (filled deck.html + figures.py + render.py → PDF) — M2/M3 mirror it (mechanics in `memory.md`).

## M2 — Generation (planned)

As-designed: `slide-gen generate [<project>]` drives `codex exec` (one shot per project) to author a deck BUNDLE at `decks/<project>/` — `deck.html` (template-styled) + `provenance.json` (per-claim source citations) + `gaps.md` (unverifiable / human-needed) + `figures.py` + `assets/*.png`. A deterministic MoonBit VALIDATOR gates every bundle (template conformance + provenance quote-grep + furigana structural guards); a project-local Python analyzer cross-checks furigana readings (hybrid). Deck CONTENT = codex's job (nondeterministic, gated behind an env flag + external `timeout`); the harness + validator are the testable MoonBit surface.

Resolved this plan (probe + research workflow + owner). Env/tool mechanics (codex exec flags, furigana tooling) → memory.md; here = decisions:
- **Gate PASSED** — `codex exec` with an explicit `--sandbox` IGNORES the interactive trust gate (probe: read-only exec on non-trusted `uas` → exit 0, replied). The "20 non-trusted projects" worry is moot; `trust_level` only gates project-local `.codex/` loading, not runtime.
- **Sandbox** = bypass (`--dangerously-bypass-approvals-and-sandbox`) — owner; the container is externally sandboxed. Harness must still direct writes to the bundle dir (bypass = no write confinement).
- **Input scope** = codex may read ANYTHING — owner (overrode both curated options); no deny-list / no input curation. Independent of that, the provenance **quote-grep** anti-hallucination check stays.
- **Furigana** = HYBRID — owner: codex writes the three tiers incl. furigana → MoonBit structural guards (`.jp`==rt-stripped-`.jpf` · `<rt>` all-kana · ruby covers kanji runs) → Python analyzer (fugashi+UniDic / SudachiPy) independently re-reads the JA; disagreements / analyzer-unknowns → `gaps.md` human-review.
- **Output** = COMMITTED deck home — owner: track `decks/<project>/{deck.html,provenance.json,gaps.md,figures.py,assets/*.png}`; gitignore only render intermediates (`_deck_full.png`, `page_*.png`).
- **Provenance contract** (research): `provenance.json` rows `{id,slide,claim,status:verified|inferred|gap,src:"path:Lstart-Lend",quote:"<verbatim>"}`; deck claims carry `data-claim="id"`. The quote must LITERALLY occur in `src` — the one cheap deterministic fabrication-catcher (line-range soft, quote hard). Prompt drives a codex self-critic DRAFT→re-verify-quotes→FINAL.
- **Section-count gotcha** (memory, re-confirmed): count `<section class="slide">` after COMMENT-STRIP — the template's line-8 header comment inflates a raw `grep -c` by 1.
- **Figures**: codex authors `figures.py` from COMMITTED, CITED numbers and runs it (bypass permits exec) → `assets/*.png`; raw-data or failed figures → a `.fig-ph` placeholder + a `gaps.md` entry (human runs it, à la the prototype's `figure_example.py`).

Units (≤200K each; gate-independent deterministic units first, then the live-codex ones):
- **M2.1 Deck structural validator** [OPEN] — `validate.mbt`, deck.html checks via targeted string-scan (MoonBit has no stdlib HTML parser): skeleton (`<!doctype>` · `<html lang="ja">` · `<meta charset="utf-8">` · `<style>` · `<body>`); comment-strip section count N≥1; each slide has one `.eyebrow` / ≥1 `.entry` / one `.pageno`; entry tiers `.jp`/`.jpf`/`.en` in DOM order; **JA-pairing** `normalize(.jp)==normalize(.jpf)` (strip tags + `<rt>` contents) — MUST core, doubles as furigana guard #1; ruby well-formed (one `<rt>` each, all inside `.jpf`, `<rt>` all-kana, ruby spans cover kanji); `.eyebrow .n` = 2-digit slide index; `.pageno` matches `^slug · n / N$`; every `<img src>` starts `assets/` and resolves on disk. Structured MUST(block)/SHOULD(warn) report. Tests: good fixture passes; seeded defects (line-8 miscount · unpaired jp/jpf · non-kana rt · missing asset · bad pageno) each caught by the right check. Accept: passes a hand-built good deck, fails each seeded defect with the correct verdict; no codex.
- **M2.2 Provenance + gaps validator** [OPEN] — extend the validator: `provenance.json` `@json` schema (each row has `status`; verified/inferred carry `src`+`quote`); **quote-grep** = each quote literally occurs in its cited src file (MUST, anti-hallucination); `src` path exists (MUST), line-range ≤ file length (SHOULD); `data-claim` cross-ref (every deck `data-claim` → a row; no verified row lacks a quote); `gaps.md` present + ≥1 structured entry. Defines + enforces the M2 output contract. Tests: fabricated-quote · dangling-claim · missing-gaps · schema-invalid fixtures each caught. Accept: catches each seeded provenance defect; no codex.
- **M2.3 Codex harness + `generate <project>` + dry-run** [OPEN] — `generate.mbt`: pure `build_codex_argv` (bypass flags, prompt via stdin) + `decks/<project>/` path resolution (create dir + `assets/`); `@process.collect_output` invocation wrapped in the `timeout` binary; failure detection (exit≠0 · missing/empty `deck.html`) → run the M2.1+M2.2 validator on the output; write-temp/rename/retry-once idempotency. Wire the `generate <project>` subcommand (validate project ∈ candidates, like `enable`). `--dry-run` prints the exact argv + resolved paths + prompt with NO codex spend. `.gitignore`: track `decks/`, ignore only render intermediates. First-draft prompt inline (refined in M2.4). Tests: argv / path / dry-run goldens; the live-codex path gated by an env flag (e.g. `SLIDE_GEN_LIVE`), mirroring M1.3's `SLIDE_GEN_BIN` e2e gate so bare `moon test` stays cheap. Accept: `--dry-run` prints the exact invocation; a gated live `generate <proj>` lands a validator-passing bundle under `decks/<proj>/`.
- **M2.4 Prompt contract + live refinement** [OPEN, live-codex spend] — author `templates/deck-prompt.md` (loaded at runtime): embed `templates/deck.html` as the style contract; three-tier entries; the `provenance.json`+`data-claim`+`gaps.md` contract with ONE verified-row + ONE gap-row few-shot; DRAFT→re-verify-quotes→FINAL self-critic; figure policy (author + run `figures.py` from committed cited numbers → `assets/*.png`; raw-data → placeholder+gaps); write the bundle to fixed paths, final message = a manifest; keep secrets/PHI out of the DECK (content guideline — owner allows READING anything, but a deck must not dump creds). Refine against real codex runs on `spine-segmentation` (richest input + a hand-authored reference deck to compare) and `rehab` (committed aggregate numbers → a real figure) until the output passes the M2.1+M2.2 validator + emits a non-trivial `gaps.md`. Accept: a live run yields a validator-passing bundle with plausible gaps; the prompt is version-controlled + documented.
- **M2.5 Furigana analyzer cross-check** [OPEN] — project-local Python helper (`tools/furigana_check.py`, uv-managed; authoring-time only, never in the MoonBit binary): read the deck's `.jp`/`.jpf`, run fugashi+UniDic (or SudachiPy — decide at impl; UniDic ≈ best analyzer CER) to independently read the JA, compare vs codex's furigana; agreements ok, disagreements / analyzer-unknowns → append structured `gaps.md` entries (term · codex-reading · analyzer-reading · human-review). Wire post-codex in the generate pipeline. Tests: the helper flags a deliberately-wrong reading and passes a correct one (fixture = the spine reference deck). Accept: flags a wrong reading to gaps, passes a correct one; wired; authoring-time only.
- **M2.6 Batch over enabled set + robustness** [OPEN] — `generate` with no arg → every enabled project (from state); per-project failure isolation (one failure ≠ abort the batch); a summary report; retry-once; timeout / partial-output handling; idempotent regenerate (overwrite). Tests: batch dispatch over a fixture enabled-set (dry-run, no live codex); failure-isolation behavior. Accept: processes every enabled project, isolates failures, prints a summary; a gated live run over ≥2 enabled projects lands validated bundles.

Sequence: M2.1→M2.2 (pure validators, no codex) → M2.3 (harness + dry-run; live path env-gated) → M2.4 (prompt + live refinement) → M2.5 (furigana analyzer) → M2.6 (batch). Gates: extend the justfile (existing `fmt`/`check`/`build`/`test` + the Python tool's lint/test + the env-gated live-codex test).
